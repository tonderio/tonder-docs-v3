## 3. How It Works (Payment Flow)

1.  The customer selects the cash payment method at checkout.
2.  The system generates a payment reference (e.g., a number or barcode).
3.  The customer takes the reference to a participating location and pays in cash.
4.  Tonder receives confirmation from the processor and notifies you via webhook.

## Payment Method Selection

 For Customers:

Available payment methods during checkout depend on:

1. **Country** - Different countries have different methods
2. **Currency** - Transaction currency affects availability
3. **Transaction Amount** - Some methods have limits
4. **Integration Model** - Your integration determines which methods are available

 For Merchants:
 
To enable payment methods:

1. **Check Coverage** - Review the [Countries Coverage](/online-payments/coverage/coverage) page
2. **Contact Support** - Request activation for desired methods
3. **Complete Onboarding** - Some methods require additional documentation
4. **Test Integration** - Verify methods work in sandbox before going live

## Important considerations

Please review these important notes regarding specific payment flows.

 ### Withdrawal Availability

Withdrawals (payouts) are currently available only in Mexico via SPEI (to a bank account or debit card). Mercado Pago withdrawals will be available soon.

 ## Cash Payment Expirations

Payment references generated for cash payments have an expiration period (e.g., 24-72 hours). Customers must pay within this window. Confirmation times vary by provider, from real-time to several hours.

 ## Refund Policies

- Cards: Standard refund processing applies.

- Bank Transfers (SPEI): Refunds are not supported for SPEI deposits.

- Cash: Refunds are not supported for cash payments.

- Mercado Pago: Standard wallet refund processing applies.
