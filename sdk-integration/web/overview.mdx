## Choosing the Right SDK

Each SDK serves a specific purpose in the payment integration workflow. Below you'll find detailed information about each SDK, including their best use cases, features, and practical implementation guidance.

### JS Inline SDK

The JS Inline SDK is designed for quick, single-transaction payments and does not directly expose methods for enrolling or managing saved cards. Its primary purpose is to provide a complete, out-of-the-box checkout UI.

**Best for:**
- Quick integration and prototyping
- Standard checkout flows
- Teams with limited frontend development resources
- Projects that prioritize speed to market

**Features:**
- Pre-built, customizable checkout UI
- Automatic form validation
- Built-in error handling
- PCI DSS compliant out of the box

### JS Lite SDK

The JS Lite SDK is designed for developers who want full control over the checkout experience. It provides a complete, out-of-the-box checkout UI that handles all the complex payment logic for you.

**Best for:**
- Custom checkout designs
- Complex payment flows
- Existing design systems
- Advanced payment method management

**Features:**
- Complete UI control
- Card enrollment and management
- Advanced customization options
- Integration with existing forms
